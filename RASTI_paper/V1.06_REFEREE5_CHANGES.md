## Summary of Changes

All referee comments have been addressed in this new version.

### 1. Softened Semi-Autonomous Claim in Section 1.1 ✅

**Original text**:
> However, it does operate in a semi-autonomous way in that if given a natural language question, it can then act autonomously within its framework to complete the task, if necessary developing its own code, tests and conclusions.

**New text**:
> The architecture is designed to support natural language queries that trigger autonomous task completion, though this capability is not demonstrated in the present test cases.

**Reasoning**: None of the six test cases demonstrate ASTRA receiving natural language input and autonomously generating code. The softened claim acknowledges this is a designed capability rather than a demonstrated one.

### 2. Fixed [htbp] Float Specifiers Visible as Text ✅

**Problem**: The [htbp] placement specifier was appearing as literal text above Figures 1-5 in the PDF.

**Solution**: Removed all `[htbp]` placement specifiers from `\begin{figure*}[htbp]` commands, changing them to `\begin{figure*}`. The rasti document class does not properly parse the placement specifier, causing it to render as visible text.

**Files affected**: 6 figures (Figures 1-6)

**Fixed instances**:
